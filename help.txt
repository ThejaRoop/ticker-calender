Ubuntu server deploy (fool-proof)
---------------------------------
1. bash scripts/setup_ubuntu.sh
2. python run_server.py doctor          # must pass before going live
3. python run_server.py list            # confirm scheduled times
4. python run_server.py run-rule popular_weekday   # smoke test
5. python run_server.py serve           # or use systemd (deploy/ticker-calendar.service)

If pip fails on curl_cffi: requirements.txt pins yfinance==0.2.54 (not 1.x).
yfinance 1.x needs Python 3.10+ and curl_cffi>=0.15.
Ubuntu setup

cd ~/ticker-calendar
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# Set your ntfy topic
export NTFY_TOPIC=test_myalerts

# Test one rule manually
python run_server.py run-rule earnings_today --time 10:00

# See all scheduled jobs
python run_server.py list

# Install as a service (edit paths/user in deploy/ticker-calendar.service first)
sudo cp deploy/ticker-calendar.service /etc/systemd/system/
sudo systemctl daemon-reload
sudo systemctl enable --now ticker-calendar
sudo systemctl status ticker-calendar

Monitor health:

cat logs/heartbeat.txt    # updated every minute if alive
tail -f logs/server.log   # job run details
Run tests
pip install -r requirements-dev.txt
python -m pytest tests/ -v
